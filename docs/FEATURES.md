# MySQL Performance Schema 监控工具 - 功能特性

## 🎯 核心功能

### 1. 实时慢查询监控

- ✅ 基于 Performance Schema 实时检测
- ✅ 可配置检查间隔（默认 10 秒）
- ✅ 可配置慢查询阈值（默认 10 秒）
- ✅ 显示完整 SQL 语句（即使查询未完成）
- ✅ 提供 KILL 命令快速终止

### 2. 详细查询信息

每个慢查询显示：
- 线程 ID 和连接 ID
- 用户和来源 IP
- 数据库名称
- 执行时间和锁等待时间
- 扫描行数和返回行数
- 当前执行状态
- SQL 语句（自动截断过长语句）

### 3. 多种通知方式

#### 钉钉机器人通知
- ✅ 支持 Webhook 推送
- ✅ 支持加签安全验证
- ✅ 格式化告警内容
- ✅ 实时推送到钉钉群

#### 邮件通知
- ✅ 支持多收件人（逗号分隔）
- ✅ 支持 SMTP 认证
- ✅ 支持 TLS/STARTTLS
- ✅ 兼容主流邮箱（QQ/163/Gmail/企业邮箱）

#### 通知特性
- ✅ 告警去重（可配置通知间隔）
- ✅ 支持同时启用多种通知方式
- ✅ 通知失败不影响监控运行
- ✅ 详细的通知日志

### 4. 多数据库监控

- ✅ 支持同时监控多个数据库
- ✅ 每个数据库独立配置
- ✅ 每个数据库独立容器
- ✅ 互不干扰，独立重启

### 5. Docker 化部署

- ✅ 完整的 Docker Compose 配置
- ✅ 多阶段构建优化镜像大小（~20MB）
- ✅ 基于 Alpine Linux 轻量级镜像
- ✅ 非 root 用户运行，安全性高
- ✅ 自动重启机制
- ✅ 日志自动轮转

### 6. 灵活配置

- ✅ 环境变量配置（12+ 参数）
- ✅ 命令行参数支持
- ✅ .env 文件管理
- ✅ 热重启应用配置

### 7. 便捷管理

#### Makefile 命令（27+ 命令）
- 服务管理：up/down/start/stop/restart
- 日志查看：logs/logs-prod/logs-tail
- 状态检查：status
- 更新部署：build/update/update-config
- 容器调试：shell-prod

#### 一键启动脚本
- 自动检查 Docker 环境
- 交互式配置引导
- 自动构建镜像
- 自动启动服务

## 💡 使用建议

1. **生产环境必须配置通知**，确保及时发现问题
2. **设置合理的阈值**，避免误报（建议 10-30 秒）
3. **配置通知间隔**，避免告警风暴（建议 3-5 分钟）
4. **部署到独立服务器**，避免与数据库"同归于尽"
5. **定期测试通知**，确保告警渠道畅通
6. **结合 Grafana**，构建完整监控体系

## 🚀 快速开始

```bash
# 1. 进入工具目录
cd mysql-monitor

# 2. 一键启动
./scripts/quick_start.sh

# 3. 配置通知（可选）
vi .env.monitor

# 4. 重启应用配置
make restart-prod

# 5. 查看日志
make logs-prod
```
